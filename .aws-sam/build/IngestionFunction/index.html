🚀 IoT Data Ingestion Pipeline

Cours : Ingénierie Big Data & IA — Architecture Cloud AWS

Date : Juin 2026

📋 Description de l'Architecture

Cette infrastructure déploie un pipeline d'ingestion de données IoT en temps réel, entièrement automatisé via AWS CloudFormation (SAM). L'architecture suit les principes Serverless et Big Data pour assurer scalabilité, haute disponibilité et faible latence.

Composants déployés :

Amazon CloudFront CDN d'ingestion — Distribution CloudFront placée devant l'API Gateway pour accélérer la collecte mondiale des données IoT (réduction latence, terminaison TLS, cache des réponses).
Amazon API Gateway Point d'entrée HTTP — Endpoint REST POST /ingest qui reçoit les payloads JSON des capteurs IoT et les transmet à la fonction Lambda.
AWS Lambda Moteur de traitement — Fonction Python 3.11 qui parse le JSON, calcule les métriques (température moyenne, count d'erreurs), stocke le payload brut dans S3 et enregistre le rapport dans DynamoDB.
Amazon S3 Data Lake brut — Bucket raw-data avec partitionnement temporel raw-zone/year=YYYY/month=MM/ pour le stockage des données IoT brutes.
Amazon DynamoDB Feature Store — Table metrics (PAY_PER_REQUEST) stockant les rapports d'exécution : ID requête, timestamp, chemin S3, température moyenne, nombre d'anomalies.
Amazon CloudFront + OAC CDN documentation sécurisé — Seconde distribution CloudFront avec Origin Access Control (OAC) pour servir le site de documentation technique depuis S3 de manière sécurisée (accès direct S3 bloqué).

🔁 Flux de données

  1. Capteurs IoT envoient requête HTTP POST vers /ingest
  2. CloudFront achemine via le CDN vers API Gateway
  3. Lambda déclenchée : parse, calcule, stocke
  4. Payload brut → S3 (Data Lake) avec partitionnement temporel
  5. Métriques agrégées → DynamoDB (Serving Layer / Feature Store)
  6. Réponse HTTP 201 renvoyée au capteur

🛡️ Sécurité

  • OAC (Origin Access Control) : le bucket S3 de documentation est inaccessible directement ; seul CloudFront peut y accéder
  • IAM Policies : chaque service a le minimum de privilèges nécessaire
  • HTTPS : toutes les communications sont chiffrées
  • Logs CloudWatch : toutes les exécutions Lambda sont tracées
